# Phase 4.5: Pattern Quality API — Implementation Plan

**Date**: 2026-04-13

## Objective

Validate and score triage patterns for quality. Ensure FMEA runbooks produce accurate, actionable diagnoses.

## Background

As we accumulate triage patterns (from runbooks, historical incidents, manual entries), we need to:
- Track pattern accuracy (does this diagnosis lead to resolution?)
- Identify low-confidence patterns
- Detect conflicting patterns
- Surface high-quality patterns for reuse

## Deliverables

### 1. Pattern Quality Metrics

Track per-pattern:
- **Accuracy**: % of times diagnosis led to successful resolution
- **Confidence drift**: Difference between predicted vs actual confidence
- **Usage frequency**: How often pattern is triggered
- **Resolution time**: Average time to resolve when pattern matches

### 2. Quality Score Calculation

```
quality_score = (accuracy * 0.4) + (confidence_accuracy * 0.3) + (recency * 0.2) + (coverage * 0.1)
```

Where:
- accuracy: 0-1 based on successful resolutions
- confidence_accuracy: 1 - |predicted_confidence - actual_outcome|
- recency: decay factor based on last usage
- coverage: fraction of similar incidents this pattern covers

### 3. API Endpoints

#### `GET /ops/patterns/quality`
List all patterns with quality scores.

#### `GET /ops/patterns/{pattern_id}/metrics`
Get detailed metrics for a pattern.

#### `POST /ops/patterns/{pattern_id}/feedback`
Record outcome feedback (success/failure).

#### `GET /ops/patterns/bottom-10`
List lowest quality patterns (needs improvement).

#### `GET /ops/patterns/top-10`
List highest quality patterns.

### 4. Pattern Registry

Store patterns in new table:
```sql
CREATE TABLE ops_patterns (
    id TEXT PRIMARY KEY,
    name TEXT UNIQUE,
    pattern_type TEXT,  -- runbook, manual, learned
    source TEXT,        -- runbook name, incident ID, etc.
    trigger_conditions TEXT,  -- JSON
    diagnosis TEXT,
    avg_confidence REAL,
    usage_count INTEGER DEFAULT 0,
    success_count INTEGER DEFAULT 0,
    last_used_at TEXT,
    quality_score REAL DEFAULT 0,
    created_at TEXT,
    updated_at TEXT
);
```

### 5. Feedback Loop

When triage completes:
1. Record which pattern was used
2. Record predicted confidence
3. Later, record actual outcome (resolved/not resolved)
4. Update pattern metrics
5. Recalculate quality score
