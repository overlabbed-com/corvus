# Phase 4.6: Graph-Powered Triage — Implementation Plan

**Date**: 2026-04-13

## Objective

Enhance FMEA triage with graph context. Use Neo4j relationships to provide richer diagnoses and root cause analysis.

## Background

Current triage is isolated per service. But failures propagate through dependencies:
- Database down → API fails → Frontend errors
- Network partition → Multiple services affected
- Shared resource exhaustion → Cascading failures

Graph-powered triage uses the Neo4j graph to:
- Find upstream dependencies that might be root cause
- Identify downstream impact
- Detect shared resources (GPU, network, volume)
- Suggest fixes based on dependency graph

## Deliverables

### 1. Enhanced Triage Context

When triaging service X:
- Fetch upstream dependencies (what X depends on)
- Fetch downstream dependents (what depends on X)
- Check health of all related nodes
- Identify shared resources

### 2. Graph-Based Diagnosis

New diagnosis types:
- `upstream_dependency_failure`: Root cause is upstream service
- `shared_resource_contention`: Multiple services competing for resource
- `cascading_failure`: Failure propagated through graph
- `network_partition`: Services can't reach dependencies

### 3. API Endpoints

#### `POST /ops/runbooks/triage-with-graph`
Enhanced triage with graph context.

#### `GET /ops/triage/{incident_id}/graph-context`
Get graph context used in triage.

#### `POST /ops/triage/root-cause-analysis`
Graph-based root cause analysis.

### 4. Runbook Executor Extensions

Add graph-aware steps:
```yaml
- name: check_upstream_health
  type: graph.upstream_health
  params:
    service: vllm-inference
    depth: 2  # Check 2 levels upstream

- name: find_shared_resources
  type: graph.shared_resources
  params:
    services: [vllm-inference, vllm-embed, litellm]
```

### 5. Root Cause Suggestions

Based on graph analysis:
- "Check database health first" (if DB is upstream and unhealthy)
- "Restart GPU services in order" (if GPU contention detected)
- "Verify network connectivity" (if multiple services affected)
