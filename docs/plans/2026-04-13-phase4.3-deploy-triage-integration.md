# Phase 4.3: Deploy Triage Integration — Implementation Plan

**Date**: 2026-04-13  
**Week**: 3 (Target completion: 2026-04-27)

## Objective

Integrate FMEA triage with GitOps deploy failures. Instead of generic error messages, deploy failures should get structured diagnosis with root cause hypotheses and remediation steps.

**Current State:**
- Deploy fails → GitHub Actions logs error → manual investigation
- No correlation between deploy failure and service state
- No automatic drift detection

**Target State:**
- Deploy fails → Corvus triage → diagnosis + remediation steps
- Automatic drift detection (declared vs running state)
- Structured deployment workflow with rollback

## Background

Deploy failures fall into predictable patterns:

| Failure Type | Symptoms | Root Cause | Remediation |
|-------------|----------|------------|-------------|
| Stale config | Container starts, crashes immediately | Compose file out of sync with secrets/env | Re-sync compose, restart |
| Slow startup | Healthcheck timeout | Service takes > expected startup time | Increase timeout, optimize startup |
| Drift detected | Running config ≠ declared config | Manual changes, config leak | Re-deploy from GitOps |
| Resource exhaustion | OOMKilled, CPU throttling | Insufficient limits | Increase resources, optimize |
| Dependency unavailable | Connection refused to upstream | Dependency not ready | Check dependency health, adjust order |
| Image pull failure | ImagePullBackOff | Registry auth, tag doesn't exist | Check credentials, verify tag |

Each pattern maps to a triage diagnosis with specific remediation steps.

### 5. Drift Detection Sweep

**File**: `src/tasks/drift_detection.py`

**Schedule**: Every 10 minutes

**Logic**:
```python
for service in all_registered_services():
    running = inspect_container(service.name)
    declared = get_declared_state(service.name)
    
    if running.image != declared.image:
        create_gap_problem(
            type="gap:coverage:config-drift",
            target=service.name,
            details=f"Image drift: {running.image} vs {declared.image}"
        )
    
    if running.healthcheck != declared.healthcheck:
        create_gap_problem(
            type="gap:coverage:config-drift",
            target=service.name,
            details="Healthcheck mismatch"
        )
```

### 6. FMEA Triage Integration

**Extend runbook executor** to handle deploy failures:

```python
async def triage_deploy_failure(
    service_name: str,
    error_message: str,
    workflow_logs: str
) -> TriageResult:
    """Triage deploy failure using FMEA patterns."""
    
    # Pattern matching
    if "OOMKilled" in error_message:
        return TriageResult(
            diagnosis="resource_exhaustion",
            confidence=0.9,
            remediation=[
                "Check container memory limits",
                "Review service memory usage trends",
                "Increase limits or optimize service"
            ]
        )
    
    if "healthcheck timeout" in error_message.lower():
        return TriageResult(
            diagnosis="slow_startup",
            confidence=0.85,
            remediation=[
                "Check service startup logs",
                "Increase healthcheck timeout",
                "Optimize initialization sequence"
            ]
        )
    
    if "stale" in error_message.lower() or "out of sync" in error_message.lower():
        return TriageResult(
            diagnosis="stale_container_config",
            confidence=0.9,
            remediation=[
                "Re-sync compose file from GitOps",
                "Clear container cache",
                "Force redeploy"
            ]
        )
    
    # Default: passthrough
    return TriageResult(
        diagnosis="unknown_deploy_failure",
        confidence=0.3,
        remediation=["Manual investigation required"]
    )
```

## Risks + Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| GitHub API rate limits | Can't fetch logs | Cache logs, exponential backoff |
| Container inspection fails | Drift detection broken | Fallback to CMDB declared state |
| False positive drift | Noise in gap problems | Threshold-based alerting (e.g., 3+ mismatches) |
| Runbook executor changes | Break existing runbooks | Backward compatible, feature flag |

## Rollback Plan

1. Disable drift detection task

3. Remove deploy step types from runbook executor
4. Revert GitHub Actions workflow changes
